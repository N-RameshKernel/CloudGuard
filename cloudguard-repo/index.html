Critical CRIT-001
S3 Bucket Public Access Enabled
arn:aws:s3:::prod-media-assets
Overview
Fix
History
Description

The S3 bucket prod-media-assets has public read/write access enabled. Any internet user can read, overwrite, or delete objects in this bucket, exposing sensitive business data.

Impact
Data breach risk · Regulatory violation (GDPR, HIPAA) · Potential ransom/deletion of objects · Reputational damage
Affected Resources
s3://prod-media-assets (2.3TB · 18,492 objects)
s3://prod-media-assets-backup
Step-by-Step Fix
1 Navigate to S3 Console → Select bucket → Permissions tab
2 Under "Block public access", click Edit and enable all 4 options
3 Review and update bucket policy to remove public principals
AWS CLI Fix
aws s3api put-public-access-block \ --bucket prod-media-assets \ --public-access-block-configuration \ "BlockPublicAcls=true,\ IgnorePublicAcls=true,\ BlockPublicPolicy=true,\ RestrictPublicBuckets=true"
Terraform Snippet
resource "aws_s3_bucket_public_access_block" "fix" { bucket = "prod-media-assets" block_public_acls = true ignore_public_acls = true block_public_policy = true restrict_public_buckets = true }
